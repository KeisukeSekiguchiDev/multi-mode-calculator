# タブUI実装テスト設計

## テスト対象
- タブのホバー効果
- タブのアクティブ状態
- タブ切り替えアニメーション
- レスポンシブ対応

## テストケース

### TC-001: タブホバー効果
**前提条件**: タブが表示されている
**操作**: マウスをタブの上に乗せる
**期待結果**:
- 背景色がtransition効果でスムーズに変化する
- カーソルがpointerに変わる

### TC-002: タブアクティブ状態
**前提条件**: 標準タブがアクティブ
**操作**: 関数タブをクリック
**期待結果**:
- アクティブタブのスタイルが適用される
- 下線インジケーターが表示される
- 非アクティブタブのスタイルが解除される

### TC-003: パネル切り替えアニメーション
**前提条件**: 標準モードパネルが表示中
**操作**: 関数タブをクリック
**期待結果**:
- 標準モードパネルがフェードアウト
- 関数モードパネルがフェードイン
- アニメーション時間は300ms以内

### TC-004: レスポンシブ対応
**前提条件**: ブラウザ幅を変更
**操作**: ウィンドウ幅を480px以下にする
**期待結果**:
- タブのフォントサイズ・パディングが調整される
- タブが適切に折り返される（必要に応じて）

### TC-005: キーボード操作（アクセシビリティ）
**前提条件**: タブにフォーカスがある
**操作**: Tab/矢印キーで移動、Enterで選択
**期待結果**:
- フォーカスリングが表示される
- キーボードで操作可能

## 合格基準
- すべてのテストケースが成功すること
- アニメーションがスムーズであること
- アクセシビリティ基準（WCAG 2.1）を満たすこと
